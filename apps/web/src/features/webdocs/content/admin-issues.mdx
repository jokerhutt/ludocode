## Overview

If the admin app fails to load, returns errors, or refuses to authenticate, the cause is usually a missing environment file or an incorrect CORS/origin configuration.

---

### Check Your Origins

The most common cause is a misconfigured URL or CORS policy.

1. Verify that `apps/admin/.env` exists and contains the correct `VITE_API_URL`.
2. Confirm the backend's `FRONTEND_ORIGINS` includes the admin app's origin (`http://localhost:5174` by default).

<Tip variant="info" title="Same Checks Apply">
  The origin and environment checks are the same as for the web app. See the
  <strong> 403 & 404 Errors</strong> article in the Troubleshooting section for
  a full walkthrough.
</Tip>

---

### Check Your Firebase Claims

If Firebase authentication is enabled, admin access requires a **custom claim** on the user's Firebase account. Without it, the backend will treat the user as a regular (non-admin) user and deny access.

Refer to the **Editing Courses** article, specifically the _Authenticating as admin_ section, for step-by-step instructions on assigning the `role: "admin"` claim.

<Tip variant="warning" title="Remember to Re-login">
  After setting a custom claim, the user must log out and log back in so that
  the updated token is issued.
</Tip>
