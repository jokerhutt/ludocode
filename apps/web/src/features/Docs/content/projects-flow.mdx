## Overview

The projects feature gives users a full code editor with multi-file support, autosave, and code execution via the Piston runtime engine. Projects are gated behind a desktop-only guard route.

---

## Project Data Model

```ts
export type ProjectSnapshot = {
  projectId: string;
  projectName: string;
  projectLanguage: LanguageMetadata; // Full language config
  deleteAt?: string; // Soft-delete timestamp
  updatedAt?: number;
  files: ProjectFileSnapshot[];
};

export type ProjectFileSnapshot = {
  id?: string; // Server-assigned ID (null for new files)
  tempId?: string; // Client-generated temp ID
  path: string; // Filename (e.g., "main.py")
  language: LanguageMetadata;
  content: string; // File content
};
```

Each project is linked to a `LanguageMetadata` object that configures the Monaco editor language mode, Piston runtime, file extension, and icon.

## The Workbench

The workbench is a 3-pane layout built on the design system's `Workbench` compound component:

1. **File Tree** (left) — navigable file list with create, rename, and delete actions
2. **Editor** (center) — Monaco Editor instance with syntax highlighting for the project's language
3. **Output** (right) — execution output with expandable log entries

The `Workbench.Pane` component provides a consistent winbar + content structure for each pane.

---

## Autosave

The `useAutoSaveProject()` hook provides debounced autosaving:

- Listens to content changes with a short debounce
- Sends the full `ProjectSnapshot` to the server
- The backend computes diffs and applies updates
- A `SaveStatusIcon` shows the current save state (idle, saving, saved, error)

---

## Code Execution

The `useRunner()` hook handles code execution through the Piston API:

1. Sends the project files and language runtime to the execution endpoint
2. Receives an `OutputPacket` or `RunnerResult`:

```ts
export type RunnerResult = {
  stdout: string;
  stderr: string;
  exitCode: number;
};
```

The output pane displays results using the `LudoLog` compound component, with expand/collapse for each execution run.

<Tip variant="info" title="Piston Runtime">
  Code execution uses the Piston engine, which supports multiple programming
  languages. Available runtimes are fetched from the external emkc.org API and
  matched to each project's `LanguageMetadata.pistonId`.
</Tip>

## File Operations

### Creating Files

New files are named `script.{ext}` where `ext` comes from the project's `LanguageMetadata.extension`. If that name exists, the suffix increments: `script-1.{ext}`, `script-2.{ext}`, etc.

### Renaming Files

<Tip variant="note" title="Extension Safety">
  Renaming a file strips or preserves the language extension automatically. For
  example, renaming `main.py` to `utils` becomes `utils.py`.
</Tip>

### Subscription Limits

The number of projects a user can create depends on their subscription tier, as defined by `UserSubscription.maxProjects`.
