## Overview

The **`@ludocode/types`** package contains every shared TypeScript type, interface, and Zod schema used across the monorepo. It exports 49 files grouped into 16 domain directories.

```text
packages/types/
├── index.ts           # Barrel re-export
├── AI/                # Chat & AI types
├── Builder/           # Curriculum snapshot types
├── Catalog/           # Course tree types
├── Completion/        # Post-lesson sync types
├── Curriculum/        # Draft schemas (Zod)
├── Exercise/          # Lesson exercise types
├── FeatureFlags/      # Feature flag definitions
├── Languages/         # Language metadata & schemas
├── Onboarding/        # Onboarding flow types
├── Piston/            # Code runner types
├── Preferences/       # Career & preference types
├── Project/           # Project & runner types
├── Static/            # Static content types
├── Subscription/      # Subscription plan types
├── User/              # User profile & auth types
└── Zod/               # Shared Zod schemas
```

## Catalog Types

These define the course hierarchy that powers the catalog tree.

```ts
interface LudoCourse {
  id: string;
  title: string;
  description: string;
  modules: LudoModule[];
  languageId: string;
  subjectId: string;
}

interface LudoModule {
  id: string;
  title: string;
  description: string;
  courseId: string;
  lessons: LudoLesson[];
  order: number;
}

interface LudoLesson {
  id: string;
  title: string;
  description: string;
  moduleId: string;
  order: number;
}
```

### Flat Course Tree

The `FlatCourseTree` type flattens nested courses into keyed maps:

```ts
interface FlatCourseTree {
  languageId: string;
  courses: Record<string, LudoCourse>;
  modules: Record<string, LudoModule>;
  lessons: Record<string, LudoLesson>;
}

type CourseTreeResponse = FlatCourseTree[];
```

## Zod Schemas

The `Zod/` directory provides runtime validation schemas used by the admin builder:

| Schema                   | Validates                      | Used In           |
| ------------------------ | ------------------------------ | ----------------- |
| **OnboardingSnapSchema** | `OnboardingSubmission` fields  | Onboarding form   |
| **CourseSnapSchema**     | Course title, description      | Curriculum editor |
| **ModuleSnapshotSchema** | Module title, order            | Module editor     |
| **LessonSnapSchema**     | Lesson title, description      | Lesson editor     |
| **ExerciseSnapSchema**   | Exercise blocks & interactions | Exercise builder  |

The `ExerciseSnapSchema` includes sub-schemas for each exercise type:

- **ClozeExerciseSchema** — validates gap items and answer arrays
- **TriviaExerciseSchema** — validates SELECT options and correct answer
- **AnalyzeExerciseSchema** — validates analysis prompts
- **InfoExerciseSchema** — validates info-only exercises (no interaction)

### Language Schema

```ts
const languageFormSchema = z.object({
  name: z.string().min(1),
  slug: z.string().min(1),
  iconUrl: z.string().url(),
  description: z.string().min(1),
  color: z.string().regex(/^#[0-9a-f]{6}$/i),
  pistonRuntime: z.string().min(1),
  available: z.boolean(),
});
```

<Tip variant="tip" title="Zod v4">
  The project uses Zod v4. Schemas can be composed with `.extend()`, `.merge()`,
  and `.pick()` to share validation logic between creation and edit forms.
</Tip>
