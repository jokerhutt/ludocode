## Overview

Ludocode lets you create and manage your own courses through the admin app. The backend handles diffing and versioning automatically, and ensures that user progress is preserved for untouched lessons.

## How a Course Works

In the admin editor, a course is a structured container that groups modules and lessons within a specific language and subject.

The hierarchy is:

- **Language**
- **Subject**
- **Course**
- **Module**
- **Lesson**
- **Exercise**

A course defines metadata such as title, description, visibility, language, and subject.  
It contains ordered modules. Modules contain ordered lessons. Lessons contain ordered exercises.

All ordering is explicit and controlled in the editor.

---

## Editing Courses

Course editing is draft-based.

All structural changes are made in a draft version of the course. This includes:

- Reordering modules
- Reordering lessons
- Adding or removing exercises
- Changing exercise interaction types
- Updating metadata

When you publish, the backend creates a new snapshot of the current draft.  
The web app always reads from the latest published snapshot.

Draft changes are isolated until explicitly published.

## Adding Languages & Subjects

Courses must belong to both a **Language** and a **Subject**.

If the language or subject you need does not exist, create it first in the admin app.

### Adding a Language

1. Open the **Admin App**
2. Navigate to **Languages**
3. Click **Create Language**
4. Provide:
   - Name
   - Slug
   - Description (optional)
   - Icon name (must match an entry in `IconRegistry`)
5. Save

The new language will now be available when creating or editing courses.

---

### Adding a Subject

Subjects are scoped to a specific language.

1. Open a language
2. Navigate to **Subjects**
3. Click **Create Subject**
4. Provide:
   - Name
   - Slug
   - Description (optional)
5. Save

The subject can now be assigned to courses within that language.

### Custom Language Icons

If you add a new language and want it to display a custom icon, you must register it in the design system.

Icons are defined in:

`packages/design-system/primitives/custom-icon.tsx`

---

#### 1. Create the Icon Component

Follow the existing pattern:

```tsx
export function JavascriptIcon({ className, color }: IconProps): JSX.Element {
  return (
    <svg
      className={className}
      xmlns="http://www.w3.org/2000/svg"
      viewBox="0 0 128 128"
    >
      <path fill={color ?? "#F0DB4F"} d="M1.408 1.408h125.184v125.185H1.408z" />
      <path
        fill={color ? "your-background-color-here" : "#323330"}
        d="M116.347 96.736c-..."
      />
    </svg>
  );
}
```

Guidelines:

- Always accept `className` and `color`.
- Use `color ?? "#HEX"` for single-color fills.
- For multi-path icons, use `color ? "your-background-color-here" : "#HEX"` where needed.
- Avoid hardcoding fills unless intentional.

---

#### 2. Register the Icon

Add it to `IconRegistry` in the same file:

```ts
export const IconRegistry = {
  Python: { component: PythonIcon, category: "language" },
  Swift: { component: SwiftIcon, category: "language" },

  Javascript: { component: JavascriptIcon, category: "language" },
} as const satisfies Record<string, IconDefinition>;
```

Important:

- The key (`Javascript`) must match the icon name used in the admin app.
- Set `category: "language"`.

---

#### 3. Use It in the Admin App

When creating or editing a language:

- Set **Icon Name** to `Javascript` (or your chosen key).

If the key does not match exactly, the icon will not render.
