## Monorepo Overview

Ludocode is organized as a **monorepo** using npm workspaces. The repository contains two applications and six shared packages, all managed from a single root.

```text
ludocode-frontend/
├── apps/
│   ├── web/            # Main user-facing application
│   └── admin/          # Internal course & curriculum management tool
├── packages/
│   ├── api/            # API path factory & fetch utilities
│   ├── design-system/  # Custom UI primitives, widgets & layouts
│   ├── external/       # Third-party UI (shadcn) & AI components
│   ├── hooks/          # Shared React hooks
│   ├── types/          # Shared TypeScript types & Zod schemas
│   └── util/           # General-purpose utilities
├── docs/               # Markdown design documents
├── package.json        # Workspace root
└── tsconfig.base.json  # Shared compiler options
```

<Tip variant="tip" title="Workspace Imports">
  All shared packages are imported via the `@ludocode/` scope — for example,
  `@ludocode/types`, `@ludocode/design-system`, or `@ludocode/api`. Path aliases
  are resolved through TypeScript `paths` mappings in each project's
  `tsconfig.json`.
</Tip>

## Web App — `apps/web/`

The primary user-facing application. Students interact with courses, complete lessons, run code projects, manage profiles, and subscribe to plans.

```text
apps/web/src/
├── main.tsx             # Entry – TanStack Router + QueryClient + providers
├── App.css              # Tailwind v4 + design-system theme imports
├── routeTree.gen.ts     # Auto-generated TanStack Router tree
├── constants/           # Route paths, navigation helpers, API config, auth
├── features/            # Feature-scoped pages, components & hooks
├── hooks/               # App-level queries, mutations, guards, context
├── layouts/             # Layout shells (Hub, Lesson, Onboarding, etc.)
├── routes/              # TanStack Router file-based route definitions
├── sound/               # Audio assets
├── styles/              # Global style overrides
├── legal/               # Legal page content (ToS, Privacy)
└── test/                # Test utilities & setup
```

| Directory      | Purpose                                                                                                                                                   |
| -------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **constants/** | Route paths (`-routes.ts`), type-safe navigation helpers (`ludoNavigation.tsx`), API config, Firebase init, environment variables, avatar data, nav icons |
| **features/**  | Self-contained feature modules — Auth, Onboarding, Hub, Lesson, Modules, Completion, Project, Subscription, Courses, AI, Error, Docs                      |
| **hooks/**     | TanStack Query definitions (`qk`, `qo`, `mutations`, `batchers`), mutation hooks (login, submit lesson, create project, etc.), guards, context hooks      |
| **layouts/**   | Structural shells that wrap route groups — `HubLayout`, `LessonLayout`, `OnboardingLayout`, `CompletionLayout`, `SubscriptionLayout`, `ProjectLayout`     |
| **routes/**    | File-based routing definitions — organized into `_app/`, `auth/`, `demo/`, `resources/` groups                                                            |

## Admin App — `apps/admin/`

The internal tool for creating and managing courses, lessons, exercises, languages, and subjects.

```text
apps/admin/src/
├── main.tsx             # Entry – Router + QueryClient
├── routeTree.gen.ts     # Auto-generated route tree
├── constants/           # Admin API, nav icons, Firebase, env, form config
├── features/            # Auth, CoursesHub, LanguagesHub, Curriculum,
│                        #   Lesson, Language, Subjects
├── hooks/               # Admin-specific queries & mutations
├── layouts/             # AdminHubLayout, CurriculumLayout, etc.
└── routes/              # File-based admin routes
```

The admin app shares the same foundational stack (React 19, TanStack Router & Query & Form) plus `@dnd-kit` for drag-and-drop exercise reordering and Monaco Editor for code editing.

## Shared Packages

### `@ludocode/api`

HTTP layer shared by both apps. Exports `createApiPaths()` for building typed endpoint URLs across 15+ API domains, typed fetch wrappers (`ludoGet`, `ludoPost`, `ludoPatch`, `ludoPut`, `ludoDelete`), and `makeIdBatcher()` for batched data loading.

### `@ludocode/design-system`

The custom component library — **primitives** (27 components: buttons, cards, inputs, icons, badges, spinners, ribbons, tabs), **widgets** (14 compound components: dialogs, menus, code preview, chatbot, file tree, lesson path, workbench), **zones** (header, footer, hero, aside, winbar), **layouts** (grid wrappers), and **styles** (design tokens, fonts, scrollbar utilities).

### `@ludocode/external`

Third-party UI components — 27 **shadcn/ui** primitives (accordion, dialog, drawer, select, etc.) and 30 **AI element** components (chat windows, prompt inputs, message renderers, canvas, model selectors).

### `@ludocode/hooks`

Shared React hooks: `useIsMobile`, `useAutoWidth`, `useHotkeys`, `useAutoScrollDown`, `useModal`, `useTab`, `useLottie`, `useTimedLottie`.

### `@ludocode/types`

All shared TypeScript types and Zod validation schemas organized by domain — User, Catalog, Exercise, Completion, Onboarding, Project, Subscription, Builder, Curriculum, FeatureFlags, Languages, Piston, Preferences, and more. Contains 49 files across 16 directories.

### `@ludocode/util`

Lightweight utility functions. Exports date formatting helpers via `dayjs`.

## Feature Directory Convention

Both apps follow a consistent feature directory pattern:

```text
features/
└── FeatureName/
    ├── Pages/           # Top-level page components
    ├── Components/      # Feature-scoped UI
    │   ├── Card/
    │   ├── Zone/        # Headers, footers, section shells
    │   ├── Group/       # Composed component groups
    │   ├── Dialog/
    │   ├── Button/
    │   └── Drawer/
    ├── Context/         # React context providers
    ├── Hooks/           # Feature-scoped hooks
    ├── Templates/       # Composite layout templates
    └── Util/            # Feature-specific utilities
```

<Tip variant="info" title="Naming Convention">
  Pages are suffixed with `Page` (e.g., `LessonPage`). Layouts are suffixed with
  `Layout` (e.g., `HubLayout`). Hooks are prefixed with `use` (e.g.,
  `useExercise`). Context providers follow `XContext` naming (e.g.,
  `OnboardingContext`).
</Tip>
