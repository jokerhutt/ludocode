## Overview

The **`@ludocode/design-system`** package contains all custom UI components shared across the web and admin apps. It is organized into five layers: **primitives**, **widgets**, **zones**, **layouts**, and **styles**.

The `@ludocode/external` package complements it with 27 shadcn/ui components and 30 Vercel AI elements.

---

## Architecture

```text
packages/design-system/
├── cn-utils.ts          # Tailwind class merging utility
├── primitives/          # 27 atomic UI components
├── widgets/             # 14 compound components
│   ├── chatbot/         # AI chatbot components
│   └── exercise/        # Exercise block renderer
├── zones/               # Page-level structural regions
├── layouts/grid/        # CSS grid wrappers
├── styles/
│   ├── theme/           # Design tokens (colors, shadows)
│   ├── components/      # Grid utility classes
│   ├── utilities/       # Scrollbar utilities
│   └── font/            # DMMono & NeueHaas font faces
└── templates/dialog/    # Pre-built dialog templates
```

## Design Tokens

The theme is defined in CSS custom properties (`styles/theme/ludo.css`):

| Token                       | Value     | Usage                             |
| --------------------------- | --------- | --------------------------------- |
| `--color-ludo-background`   | `#22273e` | App background                    |
| `--color-ludo-surface`      | `#303966` | Cards, panels, headers            |
| `--color-ludo-accent`       | `#5c6fd6` | Primary accent, active states     |
| `--color-ludo-accent-muted` | `#9eb2ff` | Secondary accent, links           |
| `--color-ludo-danger`       | `#e57373` | Error states, destructive actions |
| `--color-ludo-progress`     | `#6a7cff` | Progress bars                     |

Fonts: **NeueHaasDisplay** (headings, body) and **DMMono** (code, monospace).

## Primitives

Primitives are the smallest reusable components. Each is self-contained with no feature-specific logic.

### Core Components

| Component       | Description                                     | Variants                            |
| --------------- | ----------------------------------------------- | ----------------------------------- |
| **LudoButton**  | Primary button with 3D shadow effect            | `default`, `alt`, `white`, `danger` |
| **LudoCard**    | Card container                                  | Same variants as LudoButton         |
| **LudoInput**   | Text input with password toggle & error display | `default`, `alt`, `dark`            |
| **LudoSelect**  | Radix Select wrapper                            | `default`, `dark`                   |
| **LudoSpinner** | Centered loading spinner                        | `wide` mode                         |
| **LudoTab**     | Compound tab component (`.Group` + `.Item`)     | Active accent styling               |
| **LudoOption**  | Exercise answer option                          | `pill`, `wideSingleSelect`          |

### Display Components

| Component                     | Description                                           |
| ----------------------------- | ----------------------------------------------------- |
| **Avatar**                    | Rounded avatar with loading shimmer and accent border |
| **Badge** / **AnimatedBadge** | Icon badge with optional spring animation             |
| **CompletionBadge**           | Numbered circle — green check when complete           |
| **ProgressSummary**           | Progress bar + label (row or column layout)           |
| **CompletionRibbon**          | Diagonal corner ribbons for lesson states             |
| **IncrementingCounter**       | Animated number transition (slide up)                 |
| **StatusDot**                 | Colored circle indicator (default/ok/error)           |
| **LabelPair**                 | Simple label + value pair row                         |

### Interactive Components

| Component            | Description                                        |
| -------------------- | -------------------------------------------------- |
| **IconButton**       | Button wrapping HeroIcon with hover effect         |
| **HollowSlotButton** | Ghost-style button with active accent border       |
| **ShadowLessButton** | LudoButton without 3D shadow                       |
| **OptionInputSlot**  | Inline code-gap input with auto-width              |
| **InlineCode**       | Syntax-highlighted inline code snippet             |
| **RouterBar**        | Animated loading bar (tracks router pending state) |

### Icons

The **CustomIcon** component renders from a built-in `IconRegistry` with categorized SVG icons:

- **Languages** — Python, JavaScript, R, Swift, C#, Go, Lua
- **Exercise Types** — Cloze, Trivia, Analyze
- **Auth Providers** — Google, GitHub, Facebook
- **Status** — Error, Load, Yes (cloud icons)
- **Misc** — AICredit, Question, Commit

**HeroIcon** wraps `@heroicons/react` for standard UI icons.

## Zones

Zones are page-level structural regions that define the layout frame.

| Zone            | Description                                                              |
| --------------- | ------------------------------------------------------------------------ |
| **HeaderShell** | `<nav>` with 12-column grid. **HeaderWithBar** adds animated `RouterBar` |
| **FooterShell** | `<footer>` with 12-column grid                                           |
| **Hero**        | Accent-bordered hero section with title, subtitle, and action slot       |
| **Aside**       | Desktop-only sidebar column (hidden on mobile, sticky)                   |
| **Winbar**      | 40px top bar with surface background                                     |

## Layouts

The grid system is built on `MainGridWrapper`, which renders a full-height CSS grid with configurable row patterns:

| Pattern        | Grid Template   | Usage                           |
| -------------- | --------------- | ------------------------------- |
| `FULL`         | `1fr`           | Single-area layouts             |
| `SITE`         | `auto 1fr auto` | Header + content + footer       |
| `ONE`          | `1fr`           | Content-only (no header/footer) |
| `SITE_INVERSE` | `auto 1fr auto` | Inverted site layout            |

Supporting components:

- **MainContentWrapper** — `<main>` with scroll container
- **SubGridWrapper** — Nested grid for two-row sections

<Tip variant="tip" title="Using cn()">
  The `cn()` utility (from `cn-utils.ts`) merges Tailwind classes using `clsx` +
  `tailwind-merge`. Use it everywhere to combine conditional classes without
  conflicts: `cn("text-white", isActive && "bg-accent")`.
</Tip>
