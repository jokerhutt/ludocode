## Overview

After a user completes all exercises in a lesson, the submission goes through a **sync** phase (sending data to the backend) and then a **completion** phase (displaying results). This flow updates the user's streak, coins, course progress, and lesson status.

---

## Sync Page

When the final exercise is committed, the lesson page navigates to the **sync route** (`/sync/:lessonId`) carrying the `LessonSubmissionRequest` in router state:

```ts
router.navigate(ludoNavigation.completion.toSyncPage(lessonId, submission));
```

The `SyncingPage` calls `useSubmitLesson()`, which:

1. POSTs the `LessonSubmissionRequest` to the backend
2. Receives a `LessonCompletionPacket` in response
3. Updates the query cache (lesson status, streak, coins, course progress)
4. Navigates to the completion page with result data in search params

```ts
export type LessonCompletionPacket = {
  content: LessonCompletionResponse;
  status: LessonCompletionStatus; // "OK" | "COURSE_COMPLETE" | "DUPLICATE"
};

export type LessonCompletionResponse = {
  newCoins: UserCoins;
  newStreak: UserStreak;
  newCourseProgress: CourseProgress;
  updatedCompletedLesson: LudoLesson;
  accuracy: number;
};
```

<Tip variant="warning" title="Duplicate Prevention">
  Each submission carries a unique `submissionId`. If the same lesson is
  submitted twice, the backend returns `"DUPLICATE"` and the sync page redirects
  back to the hub without showing completion screens.
</Tip>

## Completion Pages

After syncing, the user is navigated to the **completion route** (`/completion/:courseId/:moduleId/:lessonId`) with a `CompletionSearch` object in the URL search params:

```ts
export type CompletionSearch = {
  step: "lesson" | "streak" | "course";
  coins: number;
  accuracy: number;
  oldStreak: number;
  newStreak: number;
  completionStatus: LessonCompletionStatus;
};
```

The `CompletionPage` reads the `step` parameter to determine which screen to show. The `CompletionContext` manages navigation between steps.

### Step Flow

```text
1. lesson  →  Always shown. Displays coins earned and accuracy percentage.
2. streak  →  Shown only if oldStreak < newStreak. Animated streak counter.
3. course  →  Shown only if completionStatus === "COURSE_COMPLETE".
```

Each step transitions to the next using `ludoNavigation.completion.toStreakIncrease()` and `ludoNavigation.completion.toCourseComplete()`, which update the `step` search param while preserving other values.

## Completion UI

| Component                | Purpose                                                  |
| ------------------------ | -------------------------------------------------------- |
| **CompletionPage**       | Reads `step` and renders the appropriate screen          |
| **CompletionStatsGroup** | Animated coin/accuracy display                           |
| **CompletionFooter**     | Navigation buttons (Continue / Back to Hub)              |
| **CompletionContext**    | Provides lesson/course/module IDs and navigation helpers |

The completion screens use Lottie animations for visual celebration effects.

## Cache Updates

During sync, the following query cache entries are updated:

| Data            | Update Method       | Source                                            |
| --------------- | ------------------- | ------------------------------------------------- |
| Lesson status   | `setQueryData`      | `updatedCompletedLesson` with `isCompleted: true` |
| User streak     | `setQueryData`      | `newStreak` from response                         |
| User coins      | `setQueryData`      | `newCoins` from response                          |
| Course progress | `setQueryData`      | `newCourseProgress` from response                 |
| Weekly streak   | `invalidateQueries` | Forced refetch                                    |
